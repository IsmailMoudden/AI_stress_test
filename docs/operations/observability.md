# Observability

Scenario Workbench should be observable across user workflows, AI assistance, integration boundaries, and risk engine execution.

## Future Signals

- structured application logs
- request correlation identifiers
- audit event identifiers
- scenario and scenario version identifiers
- external integration latency and failure rates
- risk engine execution status
- AI orchestration traces where governance permits
- report generation timing

## Principles

- Logs must avoid sensitive portfolio details unless explicitly approved.
- Metrics should support service reliability and workflow diagnostics.
- Audit logs and operational logs are related but distinct.
- Alerts should map to user impact and operational action.
