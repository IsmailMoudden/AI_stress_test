# UI Philosophy

The future Scenario Workbench UI should feel like institutional risk software, not a consumer AI product.

## Principles

- Dense, readable information display.
- High data-to-ink ratio.
- Fast workflows for analysts who use the tool repeatedly.
- Clear tables, filters, comparison views, and audit trails.
- No decorative gradients, excessive whitespace, or playful assistant metaphors.
- User control over every generated assumption and shock.

## Core Surfaces

- scenario drafting workspace
- evidence and historical analogue review
- assumption grid
- shock proposal grid
- override and approval workflow
- scenario comparison view
- execution status view
- report generation and export queue

## Accessibility and Usability

The UI should support keyboard-heavy workflows, predictable focus management, strong contrast, readable typography, and precise interaction states.
